The backend has shipped Benchmarking. Add a **"Benchmarking" item to the
main navigation** (with its ⓘ tooltip from the glossary — the key

## Benchmarking page (`/benchmarking`)

**Setup row.** Dataset selector (same source as Dashboard) + benchmark
source, two modes:

1. **Sector average** — dropdown fed by `GET /api/v1/benchmarks/sectors`
   (`sectors` array). If the enterprise-creation form already captured a
   sector, preselect it and pass it explicitly. Show the `source` object
   beneath the dropdown verbatim: name, version, and note — the user must
   always see that curated values are representative research/education
   figures and that advisory work should use a custom peer set.
2. **Custom peer set** — a repeatable peer form (2 minimum, 3+ suggested):
   name, revenue, EBIT, net income, total assets, total equity, total
   debt, cash, current assets, current liabilities, capex, and optional
   prior-year revenue (enables the growth benchmark). Explain optionality:
   a missing field simply excludes that peer from that KPI's average.

**Run** → `POST /api/v1/benchmarks/compare` with `{dataset_id, sector}` or
`{dataset_id, peers: [...]}`. Recompute automatically when the selection
changes (debounced). Render from the response:

- **Headline: the Benchmark Performance Index** — a large gauge centered
  on 100 (scale 50–150), the `verdict` as a pill (outperforming / in line
  with / underperforming), and the first `narrative` sentence as the
  subtitle. Tooltip on the gauge from "Benchmark Performance Index".
- **The KPI comparison table** — one row per `kpis` entry: label (ⓘ using
  the existing glossary term where present), Actual, Benchmark, and where
  `implied_label` is set, the side-by-side **Implied vs Actual** in
  currency with the signed `excess` (this is the heart of the page: "the
  sector implies EBIT of USD 165.6; actual is USD 234.6, +69.0").
  Traffic-light dot from `rag` (green/amber/red); rows with
  `status === "context"` get a neutral "context" chip instead of a light;
  `status === "no_benchmark"` rows show "no benchmark available" — never a
  number. Show `peer_coverage` as "n=X peers" when present. The published
  color rule from `rag_rule` goes in the table header tooltip.
- **Actual-vs-benchmark chart** — horizontal bar pairs (or bullet chart)
  per scored KPI, colored by `rag`. Percent-format KPIs as %, ratio as x.
- **The narrative panel** — render the `narrative` array as prose
  paragraphs under the heading "In Words". This text is generated by the
  backend from the same certified numbers; display verbatim, no editing.
- **Checkpoint badge** — the standard green badge from
  `all_checkpoints_pass`.

Errors: 404 (unknown sector) and 422 (no sector / too few peers) return a
`detail` message — show it inline near the setup row.

**Dashboard cross-link.** On the Dashboard page, add a compact card
"Benchmarking" showing just the index gauge and verdict for the selected
dataset's saved sector (call the same compare endpoint; hide the card if no
sector is set), linking to the full page.

**Enterprise-creation form.** Wire the existing sector dropdown to the API
list (`GET /api/v1/benchmarks/sectors`) instead of a hardcoded list, and
include the chosen value as `company.sector` in the dataset JSON when
creating via direct entry — the backend accepts it and will warn (not
reject) if a sector has no curated benchmark.
